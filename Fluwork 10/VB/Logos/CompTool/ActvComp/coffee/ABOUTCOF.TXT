
========>
========> Aplicación de ejemplo Coffee 
========>

(Coffee2.vbp, CoffWat2.vbp, MTCoffee.vbp, XTimers.vbp)

NOTA: Seleccione Ajuste de línea en el menú Edición para ver este archivo más fácilmente.
 
Coffee consta de un cliente, CoffWat2, y dos componentes ActiveX (servidores OLE), Coffee2 y MTCoffee. Juntos, estos tres proyectos demuestran lo siguiente:

· Notificaciones asíncronas mediante eventos (Coffee2).
· Notificaciones asíncronas mediante métodos de devolución de llamada (Coffee2).
· Subprocesos múltiples (MTCoffee).

XTimers.vbp es un proyecto que proporciona un cronómetro de código utilizado por Coffee2 y MTCoffee.

Ésta es una versión expandida del proyecto desarrollado en los procedimientos paso a paso del capítulo 3, "Creación de un componente EXE ActiveX", de Creación de componentes ActiveX. Para obtener más información acerca de las notificaciones asíncronas y los subprocesos múltiples, consulte el capítulo 8, "Generación de componentes de código".

Para ejecutarlo

Aunque no contiene mucho código, este ejemplo es difícil de ejecutar, ya que demuestra componentes fuera de proceso. Para la depuración, un componente fuera de proceso debe ejecutarse en otra instancia distinta de Visual Basic.

Además, el ejemplo demuestra características (cronómetros de subprocesos múltipes y de código) que no se pueden demostrar, o son peligrosas de ejecutar, en el entorno de desarrollo. 

1. Cargue XTimers.vbp en Visual Basic. En el menú Archivo, seleccione Generar XTimers.dll para convertir el proyecto en un archivo .dll.

IMPORTANTE: NO ejecute ahora XTimers en el entorno de desarrollo. XTimers debe compilarse porque utiliza las API de Windows para un objeto cronómetro de código.

2. Cargue MTCoffee en Visual Basic. En el menú Proyecto, seleccione Referencias para abrir el cuadro de diálogo Referencias. Seleccione XTimers y haga clic en Aceptar para establecer una referencia a la DLL compilada.

3. En el menú Archivo, seleccione Generar MTCoffee.exe para convertir el proyecto en un archivo .exe.

IMPORTANTE: NO ejecute ahora MTCoffee en el entorno de desarrollo. Necesita generar el proyecto porque el entorno de desarrollo no acepta múltiples subprocesos de ejecución; si ejecuta MTCoffee dentro de Visual Basic, no presentará el comportamiento de subprocesos múltiples.


4. Cargue Coffee2.vbp en Visual Basic. En el menú Proyecto, seleccione Referencias para abrir el cuadro de diálogo Referencias. Seleccione XTimers y haga clic en Aceptar para establecer una referencia a la DLL compilada.

5. Presione CRTL+F5 para ejecutar el proyecto Coffee2.

Utilice CRTL+F5 en lugar de F5 cuando ejecute un componente fuera de proceso para asegurarse de que todos los errores de compilación se detectan antes de que el componente proporcione objetos al cliente. (Consulte "Creación de un componente EXE ActiveX" en los Libros en pantalla.) 

6. Inicie otra instancia de Visual Basic y cargue CoffWat2.vbp. En el menú Proyecto, seleccione Referencias para abrir el cuadro de diálogo Referencias. Seleccione Coffee2 y MTCoffee y, después, haga clic en Aceptar para establecer referencias al proyecto Coffee2 en ejecución y al archivo MTCoffee.EXE compilado.

7. Presione F5 para ejecutar el proyecto.
 
Después de ejecutar la aplicación de ejemplo y observar el comportamiento de MTCoffee cuando asigna un subproceso distinto para cada objeto Coffee, puede que le resulte instructivo volver a compilar MTCoffee después de cambiar el subproceso múltiple a un grupo de subprocesos circular. (Encontrará esta opción en el cuadro Ejecución desatendida, en la ficha General del cuadro de diálogo Propiedades del proyecto.) Cuando utilice un grupo de subprocesos circular, puede observar el bloqueo y el uso compartido de datos entre distintos objetos del mismo subproceso.

También puede compilar Coffee2.exe y CoffWat2.exe, y ejecutar múltiples clientes para observar el uso compartido de los objetos de notificación asíncrona CoffeeMonitor y CoffeeMonitor2.

Con múltiples clientes, el grupo de subprocesos circular significa bloqueo y uso compartido de datos entre los objetos Coffee en uso por parte de diferentes clientes.

Para entender lo que ocurre en este ejemplo, consulte "Generación de componentes de código", en Creación de componentes ActiveX de los Libros en pantalla.

Para agregar documentos como éste a proyectos de Visual Basic, active "Agregar como documento relacionado" en el cuadro de diálogo Agregar archivo.
